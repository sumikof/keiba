# 競馬買い目検討

引数で指定されたレース名（$ARGUMENTS）の直近の該当レースについて、`netkeiba-scraper` スキルを使って情報を収集し、買い目を検討してreportsディレクトリにレポートを作成する。

## 作業手順

### 1. 直近レースの特定
`netkeiba-scraper` スキルを使い、今日・直近の開催レース一覧を取得する。「$ARGUMENTS」に一致するレースを見つけ、レースIDを特定する。

### 2. 出走表・オッズの取得
`netkeiba-scraper` スキルを使い、対象レースの出走表と単勝・複勝・3連複オッズを取得する。

### 3. 全出走馬の過去成績取得
`netkeiba-scraper` スキルを使い、出走全頭の馬情報・過去成績を取得する。

### 4. 全頭評価
以下の観点で各馬を100点満点でスコアリングする：
- 近走成績（着順・タイム・上がり3F）
- 距離・コース適性
- 騎手・調教師
- 人気（オッズ）との乖離（穴馬候補の発掘）
- 血統・馬場状態適性

### 5. 三連複フォーメーション買い目の構成
- 軸馬（上位評価）＋ 相手馬（中・低評価の穴馬含む）でフォーメーション構成
- 点数 × 100円が **1万円以内** に収まるよう調整
- **穴馬を1頭以上** 必ずフォーメーションに組み込む

### 6. レポート作成
`reports/yyyymmdd_<レース名>.md` 形式でレポートを作成する。内容：
1. レース概要（レース名・開催日・競馬場・距離・条件）
2. 全頭評価表（馬番・馬名・評価点・評価コメント）
3. 三連複フォーメーション買い目と点数・金額
4. 総投資額と期待値の考察
5. 穴馬の注目理由
