## プロジェクト概要

日本中央競馬（JRA）の買い目を検討するプロジェクト。netkeibaからレース情報を取得し、資産最大化を目指した馬券購入戦略を立案する。

## 目標・制約

- 予算上限: **1万円**
- 中心馬券: **三連複フォーメーション買い**
- フォーメーションには必ず**穴馬（人気薄）を含める**

## 作業フロー

1. `netkeiba-scraper` スキルを使ってレース情報を取得
2. 対象レースの出走表・オッズを取得
3. 全出走馬の過去成績を `netkeiba-scraper` で取得
4. **全頭評価**: 各馬に評価理由と点数（スコア）を付ける
5. 評価を基に三連複フォーメーションの買い目を組み立て、1万円以内に収める
6. 買い目レポートをreports/に作成する。レポート名はyyyymmdd_<レース名>.md

## 全頭評価の方針

各馬を以下の観点で評価し、100点満点でスコアリングする:

| 評価項目 | 配点 | 評価内容 |
|----------|------|---------|
| 同条件重賞実績 | 25点 | 本番と同じ競馬場・距離の重賞（特にGI）での着順 |
| 競馬場別適性 | 20点 | 対象競馬場でのコース連対率（出走なしは未知数として−3点） |
| 近走成績 | 15点 | 直近の着順・内容。ただし本番とコース条件が異なる場合は割り引く |
| 前走凡走の理由分析 | 10点 | 凡走理由を分類し、巻き返し可能性を評価（下記参照） |
| 騎手・調教師 | 15点 | 騎手の実績・調教師の管理能力 |
| 血統・馬場状態適性 | 10点 | 血統のコース・馬場適性 |
| 人気（オッズ）との乖離 | 5点 | 穴馬候補の発掘（主に買い目構成段階で活用） |

### 前走凡走の理由分類

前走で着外だった馬は凡走理由を以下の4分類で判定し、評価への影響を調整する:

| 分類 | 定義 | 本番への影響 | 判定基準 |
|------|------|-------------|---------|
| A. 能力的限界 | 格上挑戦で力不足 | 大幅減点 | 格上レースで見せ場なし、タイム差大 |
| B. コース不適 | 得意でないコースでの凡走 | 減点小（巻き返し可） | 特定コースでのみ凡走、他コースでは好走のパターン |
| C. 展開不利 | ペースや位置取りが合わなかった | 中程度の減点 | 不利があった、極端なペースだった |
| D. 状態不良 | 馬体重急変、間隔が詰まっている等 | 条件付き減点 | 馬体重の大幅増減、明らかな調子落ち |

### 競馬場別適性の判定

各馬の対象競馬場での成績を集計し、コース連対率で判定する:

```
コース連対率 = (対象競馬場での1着回数 + 2着回数) / 対象競馬場での出走回数
```

| コース連対率 | 判定 | 加点 |
|-------------|------|------|
| 80%以上 | コース巧者 | +15〜20点 |
| 50〜79% | 適性あり | +5〜14点 |
| 30〜49% | 普通 | ±0点 |
| 30%未満 | 不適 | −5〜10点 |
| 出走なし | 未知数 | −3点 |

## 買い目戦略

- **1頭軸フォーメーションをメイン**に予算の70〜80%を配分（軸馬が来た時に相手を広くカバー）
- 2頭軸固定は全体の15%以下に抑えてリスク分散
- 点数 × 最低購入金額（100円）が1万円以内に収まるよう調整
- 穴馬を1頭以上必ずフォーメーションに組み込む
- 大穴展開に10〜15%を配分してローリスクで波乱を拾う
